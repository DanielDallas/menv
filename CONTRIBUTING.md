# Contributing to Menv

Menv is built for the community. If you find a bug or want to improve the logic, we welcome your contributions.

## How to Contribute

1.  **Fork the Public Repository**: Create your own copy of the [public Menv repository](https://github.com/DanielDallas/menv).
2.  **Clone the Repository**: Clone your fork locally.
3.  **Create a Branch**: Create a new branch for your feature or bug fix.
4.  **Make Changes**: Implement your changes and ensure they follow our coding standards.
5.  **Write Tests**: Add or update tests as necessary.
6.  **Run Tests**: Ensure all tests pass by running `npm run test`.
7.  **Submit a Pull Request**: Submit a PR to the `main` branch of our public repository.

## Coding Standards

- Use TypeScript for all source code.
- Follow the existing code style (ESM, CAC for CLI).
- ensure all public commands are well-documented in the README.
- Write clear, professional commit messages.

## Issue Reporting

If you find a bug or have a feature request, please open an issue on the public repository with a clear description and any relevant context.

Thank you for your contributions!
